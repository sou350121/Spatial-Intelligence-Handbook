# 标定 — 多相机、IMU-相机、在线自标定 / Calibration in the Field

**TL;DR:** 出厂标定一定会在现场漂掉。手册级综述很少写这件事，但工程上：每一颗螺丝拧紧的扭矩、每一次温度循环、每一次跌落都在改你的外参与时间偏置。这页给的是一个有立场的工作流：用 Kalibr 做初始标定，用硬件触发或 PTP 同步时间，把「运行中自标」当作一等公民设计进栈，而不是当作兜底。

## 1 · 为什么出厂标定一定会漂

任何把传感器装在结构上的载体，外参都会随以下几件事变化：

| 因素 | 量级（`UNVERIFIED`） | 备注 |
|---|---|---|
| 温度循环（-20 °C → 60 °C） | 旋转 0.05–0.3°，平移 100–500 µm | CFRP 或铝合金骨架的常规变形 |
| 振动（电机、螺旋桨、地面冲击） | 旋转 0.01–0.1° / 小时累积 | 紧固件松动是主因，不是材料蠕变 |
| 软着陆 / 碰撞 | 旋转 0.1–2°，平移 1–10 mm | 一次硬触地就足以毁掉一次出厂标定 |
| 长期老化 | 旋转 0.05° / 月 `UNVERIFIED` | 主要来自胶水蠕变与紧固件预紧力衰减 |
| 镜头自身的对焦 / 防抖位移 | 内参漂移，不只是外参 | 消费级相机带 OIS 时尤其严重 |

对一个 1 m 基线的双目，0.1° 旋转误差在 10 m 远处对应**约 1.7 cm 横向偏差**；对一个 0.1 m 基线的双目则被放大十倍。这是「标定一旦漂，就直接坏视差三角化」的物理原因。

「出厂标定一次顶一辈子」是错觉，且是非常常见的错觉。

## 2 · 三层标定的标准分工

| 层 | 何时做 | 工具 | 输出 |
|---|---|---|---|
| **L1：初始 / 工厂** | 出厂；机械改动后；大故障后 | Kalibr、ROS camera_calibration、AprilTag 标定板 | 内参、外参、时间偏置初值 |
| **L2：上电 / 任务前** | 每次开机；任务起飞前 | 静态自检 + 短运动 + Kalibr 重算 | 验证 L1 仍在容差内；漂太多就报警 |
| **L3：运行中（在线自标）** | 整个任务 | EKF / 因子图把外参 + 时间偏置当作状态联合估计 | 实时跟踪缓慢漂移 |

学术综述常常停在 L1。工业部署主要靠 L2 + L3。**这一页只在 L1 上有用的内容，多半在工程上没用。**

## 3 · Kalibr —— L1 的事实标准（与坑点）

Kalibr（Furgale et al., ETH/ASL）是多相机 + IMU 联合标定的开源事实标准。基本工作流：

```
1) 打印 AprilGrid 标定板（已知大小 / 已知 tag 编码）
2) 录一段 ≥60 秒、包含三轴充分激励的 rosbag
   ├─ 6-DoF 平移 + 旋转都要走
   ├─ 标定板必须在每个相机的视野里占多个深度
   └─ IMU 噪声要事先用静止数据做 Allan 方差
3) 跑 kalibr_calibrate_cameras 拿多相机内 / 外参
4) 跑 kalibr_calibrate_imu_camera 拿 IMU-相机外参 + 时间偏置
5) 用一段未参与训练的数据做交叉验证
```

常见坑（按出现频率排）：

- **没有跑 Allan 方差**就用厂家默认 IMU 噪声 → 因子图协方差错，后续 VIO 全偏。
- **激励不充分**（运动太斜或太单一轴）→ 某些自由度不可观，外参错得很难发现。
- **标定板尺寸 / 间距错** → 几何全错但数值收敛，看起来对，跑起来烂。
- **快门类型未声明**（global vs rolling shutter）→ rolling shutter 当作 global 处理会引入时空耦合误差。
- **温度差异** → 在 20 °C 标定，到 5 °C 户外飞，热漂移没被建模。

## 4 · IMU-相机时间同步 —— 大多数 VIO 失败的真正原因

VIO 论文报告的高精度数字几乎都假设了**严格时间同步**。真实世界三个等级：

| 等级 | 实现 | 时钟偏置（`UNVERIFIED`） | 漂移 |
|---|---|---|---|
| **A. 硬件触发同步** | 一颗 MCU 同时触发相机曝光与 IMU 采样 | &lt;0.1 ms | 零（同源时钟） |
| **B. PTP / gPTP（IEEE 1588）** | 网络时间协议，硬件辅助 | 1–10 µs | 网络抖动主导 |
| **C. 软件时间戳 + 在线估计** | 各设备本地时钟，由 VIO 自身估计偏置 | 1–10 ms（初始） | 随温度 / 晶振漂移 |

C 是最常见的部署形态（USB 相机 + 串口 IMU 是消费 / 研究典型）。它的代价是：

- 必须在因子图里把 `td`（相机与 IMU 时间偏置）当作待估状态；Kalibr 也估，VIO 也估。
- 在高加速度或高角速度下，几毫秒的时间偏置就会把 IMU 预积分变成错误位姿。
- 高动态平台（无人机、人形）几乎一定要 A 级硬件触发；地面机器人 C 级常常能凑合。

经验上：**Skydio 级别的飞行品质需要 A**；**消费级 ROV 用 C 就能跑**；**工业 AGV 看负载和速度**。

## 5 · 运行中自标定 —— 一等公民设计

「运行中也能改外参」常被当作兜底功能，实际上应作为一等公民设计进栈。两种常见做法：

### 5.1 把外参 / 时间偏置纳入状态

EKF 或因子图把 `T_cam_imu`（6-DoF 外参）和 `td`（时间偏置）当作待估状态。优点：和 VIO 状态联合最优。缺点：可观性条件苛刻 —— 需要持续的 6-DoF 激励，纯直线匀速运动会让外参变得不可观。

### 5.2 触发式重标（hot-recal）

正常飞行用 L1 外参；当残差统计（e.g., reprojection error chi-square）触发阈值，进入一段「校准模式」运动，专门做激励充分的动作再回到任务。Skydio 类商用栈据信用这种模式 `UNVERIFIED`。

### 5.3 标定漂移监测指标

| 指标 | 含义 | 触发阈值（`UNVERIFIED`） |
|---|---|---|
| Reprojection error chi-square | 投影残差是否符合协方差 | 持续 >2× 期望 = 标定可疑 |
| 双目视差一致性 | 立体匹配点与三角化深度的一致性 | 不一致点 >5% 持续 60 s = 触发 |
| IMU pre-integration residual | 预积分位姿与视觉位姿的差 | 持续单向偏置 = 时间偏置漂 |
| 多相机闭环误差 | 多相机系统的闭环几何是否自洽 | 闭环 >X cm = 外参漂 |

## 6 · 跨载体的小差异

| 载体 | 主要标定挑战 |
|---|---|
| **操作（桌面）** | 工作距离短，外参对精度敏感；好处是温度稳定，振动小 |
| **地面移动（AGV）** | 振动中等；常常长时间运行，热漂移要建模 |
| **低空（无人机）** | 螺旋桨高频振动 + 着陆冲击是主敌；硬件触发几乎是必需 |
| **AD** | 多相机基线长（车宽），振动 + 热双重严重；量产栈普遍 L3 自标 |
| **人形 / 腿足** | 关节运动改变光学链路，标定要包含手臂 / 头部连杆模型 |
| **水下** | 折射在 housing port 处必须建模；不是标准针孔模型 |

## 7 · 2-year outlook + 可证伪预测

工程方向有两条在动：

1. **学习驱动的在线自标** —— 一个小网络从图像流直接预测外参修正项，不需要标定板。早期工作已出现，但鲁棒性差。
2. **AprilTag-free L1 标定** —— 用自然场景特征 + bundle adjustment 完成出厂级标定，把对标定板的依赖去掉。

**可证伪预测：** 到 2027-12 前，会出现一个开源工具能在 ≤30 秒任务起飞前自然场景动作中完成相机 + IMU 外参 + 时间偏置标定，精度接近 Kalibr。如果到那时点没有，工程界仍会被「先拿出标定板」绑架。
