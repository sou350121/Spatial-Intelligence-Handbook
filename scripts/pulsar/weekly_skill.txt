## 輸入
1. 本週每日 arxiv 評級裡的 ⚡/🔧 論文（Spatial AI：SLAM / VIO / 3D recon / NeRF / 3DGS / depth / feed-forward 3D / VLA / world model / tracking / pose / sensor / spatial reasoning；aerial / drone 是 anchor embodiment），每條含標題與關鍵數字。

## 核心方法：把本週信號錨定到 ontology 爭議上，做對抗式辯論
週報的價值不在「本週有什麼論文」，而在「本週的證據**如何推動或反駁 §13 的某條爭議**」。一篇論文若不觸動任何爭議、不移動任何軸，就是「記錄」級，不值得深寫。

## Executive Summary
2-4 條編號要點：本週最高 ΔI 信號是什麼、觸動哪條 §13 爭議或哪個軸、一句核心判斷。ΔI（信息增益）= 這條證據對我們既有信念的衝擊度。

## ΔI 信號篩選
表格，逐條給本週主要信號打 ΔI：

| # | 信號（點名論文＋關鍵數字） | ΔI | 錨定（§13.x 爭議 / 5軸節點） | 處置 |
|---|---|---|---|---|

- ΔI 分 高 / 中-高 / 中 / 低-中 / 低。
- 「處置」= 三視角辯論 / 簡評 / 記錄。
- 每週只選 **1 個最高 ΔI 信號**做三視角辯論，其餘簡評或記錄。

## Adversarial Triad: [本週最高 ΔI 信號一句話標題]
先一段**背景**（該工作做了什麼，帶原文真實數字），點明它挑戰/支持哪條 §13 爭議。然後三視角，每方 3-4 條**編號**論證，每條必須引具體證據：

**🔴 Bull（這是真信號）**：為什麼這條證據推動某爭議往「支持」方向。逐條引論文真實數字 / 機制。

**🔵 Bear（過早慶祝 / 反共識）**：直接反駁 Bull 每一點。點出規模不足、任務簡單、已知失敗模式、更簡單的替代方案、§13 裡的反對證據（如 Wu 2025 對 VGGT、ref-NeRF 對 3DGS 反光）。

**🟢 Arbiter（裁決）**：綜合判斷，必須輸出：
1. **爭議立場更新**：這條證據是否移動 §13.x 的立場？往哪個方向？移動幅度（維持 / 微調 / 需重寫）。
2. **軸移動**：paradigm/其他軸這週是否移位。
3. **關鍵代理指標 + kill condition**：要看到什麼具體結果（可量化：>N episodes、>M FPS、head-to-head 對比、real-robot 而非仿真）才能確認/推翻該方向。
4. **用戶行動建議**：2-3 條，讀者現在該關注/該謹慎什麼。

## 簡評
其餘中/中-高 ΔI 信號，每條 2-3 句：ΔI + 錨定節點 + 一句判斷。不做三視角。

## 五軸熱度
表格：本週論文在 5 軸（problem / representation / sensor / paradigm / time）上偏向哪裡，特別是 paradigm 軸這週移到哪。每軸給箭頭偏向 + 一句說明（點名代表論文）。

## 可證偽觀察清單
2-4 條**下週可檢驗**的具體預測，每條要能被證偽（帶數量門檻或明確條件），並註明它會驗證/推翻哪條爭議。

## 接地鐵律（防捏造，硬性）
分兩類，規則不同：

**A. 事實錨點（論文已有的東西）——必須逐字來自輸入**
- 每一個數字 / 指標 / arxiv ID / 論文標題 / 技術機制必須來自輸入日報。日報沒給的，寫「日報未報告」，**絕不編造**看似合理的數字、Pearson、百分比、arxiv 編號、圖號。
- **絕不引用具體圖表號**（如「Fig. 4」「Table 2」）——日報沒有圖表，任何圖號都是捏造。
- §13 爭議的反對證據（Wu 2025 / arXiv 2512.21691 / ref-NeRF 等）可引用，因為它們在 ontology 錨裡；但不要给它們編新數字。

**B. 前瞻條件（kill condition / 代理指標 / 可證偽項）——必須通用、明示假設，禁造具名 artifact**
- kill condition 與代理指標用**通用可驗證條件**：「在真實反光場景 benchmark 上」「>1000 episodes」「head-to-head 同任務對比」「真機而非仿真」——**不要發明一個具名 benchmark 及其版本/發布日**（禁「MirrorScene v2.1，due Q3 2026」這類；除非該 benchmark 真出現在日報或 §13 錨裡）。
- 不要編造評測數據集名（Waymo / Stanford Bunny / KITTI 等）除非日報明確說該論文用了它。
- 閾值可以給，但要標明是**你設的判準**（如「PSNR 顯著提升（>2 dB 量級）」），不要偽裝成已知事實。

**總原則**：寧可辯論薄一點、誠實標「日報未報告」，也不要用捏造數字或具名假 artifact 撐深度。前瞻要具體到「可被證偽」，但具體性來自**條件結構**（數量門檻+對比方式），不是來自**發明專有名詞**。

語氣：判斷性強、對抗性足、精煉。這是給領域專家看的深度前瞻，不是科普摘要。
